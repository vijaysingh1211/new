SAP Hybris (SAP Commerce Cloud):-

SAP Hybris, now known as SAP Commerce Cloud, is an enterprise-level e-commerce platform used by organizations to build and manage online stores, B2B marketplaces, B2C websites.

Diff. btw B2B and B2C?

B2B:- B2B stands for Business-to-Business, where a company sells products or services to another company. It usually involves bulk orders.
EX:-Salesforce, Boeing.

B2C:- B2C stands for Business-to-Consumer, where a company sells directly to individual customers. It typically has fixed pricing, smaller orders.
Ex- Amazon, Nike, Netflix.

SAP Hybris, now called SAP Commerce Cloud, is an enterprise e-commerce platform that helps organizations build and manage online commerce solutions. It provides features such as product management, customer management, order processing, content management, search, and multi-channel commerce. The platform follows a layered architecture consisting of presentation, service, and persistence layers. It also provides tools like Backoffice, HAC, Impex, Solr, and OCC APIs for administration and development.


Development Environment Setup:-

Development Environment Setup is the process of installing and configuring all the software required to develop and run a SAP Commerce Cloud application locally.

COMNDS:-
setantenv.bat (Sets Java path, Sets Ant path, Configures build environment)
ant clean all (Deletes old generated files, Compiles Java classes, Generates model classes)
hybrisserver.bat (Initialization (first time) OR Update (existing system),  Starts the server(Tomcat))

Server starts on: http://localhost:9001

Q. What is the purpose of local.properties?
It stores environment-specific configurations such as database details, ports, Solr configuration, and other application settings.


--> To set up a SAP Commerce development environment, we install Java, Apache Ant, a supported database, and an IDE such as IntelliJ. We extract the SAP Commerce package, configure database settings in local.properties, execute setantenv.bat, and build the project using ant clean all. After the build is successful, we start the server using hybrisserver.bat and access HAC for initialization or updates. The main folders are platform, custom, and config, where platform contains framework files, custom contains project extensions, and config contains application configuration files.



Data Modeling:-
Data Modeling is the process of defining and managing business entities and their relationships in SAP Commerce.

Examples:
Product
Customer
Order
Cart
Address
Category

In SAP Commerce, data modeling is primarily done using the items.xml file.

What is items.xml?
items.xml is the configuration file where we define:
Item Types
Attributes
Relations
Collections
Enumerations

Ex:-<itemtype code="Student" extends="GenericItem">
    <attributes>
        <attribute qualifier="studentId" type="java.lang.String">
            <modifiers optional="false"/>
            <persistence type="property"/>
        </attribute>

        <attribute qualifier="studentName" type="java.lang.String">
            <persistence type="property"/>
        </attribute>
    </attributes>
</itemtype>

This creates a new type called Student.


Important Components
1. Item Type
An Item Type is a custom model (like a class in Java) used to store data in the database.

2. Attributes
Attributes are properties of an item type.
Example:
<attribute qualifier="name" type="java.lang.String"/>

Here:
qualifier = attribute name
type = data type


3. Persistence
Defines how data is stored.

<persistence type="property"/>
Most commonly used.
Stores data in the database column.


Types of Item Types

GenericItem
For custom entities that are not products or users.
EX:(Student, College, Department, Course)
<itemtype code="Student" extends="GenericItem">

Product
For anything that will be sold in the store.
<itemtype code="BookProduct" extends="Product">
Example: Book, Mobile, Laptop

User
For customers, employees, administrators.
<itemtype code="Employee" extends="User">


Modifiers
Modifiers control attribute behavior.

Mandatory Attribute
<modifiers optional="false"/>
Value must be provided.
Example: <attribute qualifier="employeeId">
Employee ID cannot be empty.

Read Only
<modifiers read="true" write="false"/>
Cannot be modified.

Unique
<modifiers unique="true"/>
No duplicate values allowed.
Example: <attribute qualifier="email">
Every email must be unique.



Relations:
Relations connect two item types.
Example:
Student ↔ College
Employee ↔ Department
Product ↔ Category

One-to-Many Relation
One department can have many employees.
<relation code="DepartmentEmployeeRelation">
Example:
One Department
Many Employees

Many-to-Many Relation
Example: Students and Courses
One student can enroll in many courses.
One course can have many students.

One-to-One Relation
Example: Person ↔ Passport
One person has one passport.

Collection Types:
Collections allow storing multiple values of the same type in a single attribute.
Example:
A student can have multiple skills.
<collectiontype code="SkillList"
                elementtype="java.lang.String"
                type="list"/>

Use it:
<attribute qualifier="skills" type="SkillList">
    <persistence type="property"/>
</attribute>

Enumeration Types:
Enumerations (Enums) store a fixed set of values.
Example: <enumtype code="Gender">
    <value code="MALE"/>
    <value code="FEMALE"/>
     </enumtype>
